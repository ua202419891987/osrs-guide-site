🎮 RuneLite GPU Plugin Crashing & Auto-Disabling — Fix Guide 2026

The RuneLite GPU plugin is what turns Old School RuneScape from a blurry CPU-rendered game into a smooth, high-FPS experience. When it crashes or silently switches itself off, the game drops to software rendering and stutters badly. This guide explains exactly why it happens and how to fix it on NVIDIA, AMD, and Intel hardware.

🎮 Level: Novice — RuneLite Client Help · 12 min read · Updated July 2026

⏱️ Quick Verdict — 30-Second Read

  • 📌 RuneLite auto-disables the GPU plugin when the render context fails — the game keeps running on the slower CPU/software renderer, so nothing is broken.
  • 📌 90% of crashes are driver problems, not the plugin — update your NVIDIA/AMD/Intel GPU driver before changing anything else.
  • 📌 Turn OFF heavy settings on weak hardware — MSAA anti-aliasing, anisotropic filtering, and increased memory buffers are the usual culprits.
  • 📌 OpenGL is the safe default — only enable Vulkan if your drivers are current and you want extra FPS.
  • 📌 Suspect a conflict? Launch RuneLite in Safe Mode (hold Shift at startup) to isolate the bad plugin.

1. 🔧 Why the GPU Plugin Turns Off or Crashes

RuneLite's GPU plugin replaces the game's default software renderer with a hardware-accelerated one using OpenGL (the default) or Vulkan (an opt-in backend). Instead of your CPU drawing every frame, your graphics card does it — which is why the GPU plugin is responsible for almost all of RuneLite's performance gains.

When something goes wrong with that hardware pipeline, RuneLite protects you by automatically disabling the GPU plugin and dropping back to the built-in software renderer. You'll usually see a message in the chat box or client log such as "GPU plugin disabled due to an error" or "The GPU plugin failed to start". The game does not close — it just runs slower.

💡 First Question to Ask Yourself

Did the crash start right after a Windows / driver / RuneLite update, or did you just change a GPU plugin setting? If it followed an update, the driver is the prime suspect. If it followed a settings change, revert that setting first — it's the fastest fix.

2. ⚡ GPU Plugin Settings That Prevent Crashes

Open the GPU plugin config (RuneLite → Configuration → search "GPU"). These are the settings that most often cause crashes on weaker or integrated hardware, and the safe values we recommend for a stable session:

SettingSafe ValueWhy
Use VulkanOFF (use OpenGL)Vulkan is faster but crashes on many drivers, especially AMD
Anti-aliasing (MSAA)Off, or 2x max4x/8x MSAA eats VRAM and crashes integrated GPUs
Anisotropic filteringOff on integrated, 2x-4x on dedicatedHigh filtering drains memory buffers quickly
Sampling modeNearest (or Bilinear)Nearest is the most compatible with old drivers
True colorOff if crashingUses extra framebuffer memory
UI scaling1.0 (or match your screen)Non-integer scaling can glitch the context
FPS target / Unlock FPSCap at 60 (or your monitor Hz)Uncapped FPS overheats laptops and destabilizes drivers
Stretched modeUse GPU plugin's own stretch onlyExternal Stretched Mode plugin conflicts with GPU
Increase texture memory limitLeave default unless you know whyToo-high limits can exceed what your GPU allows

① The 30-Second Stability Reset

If you just want it to stop crashing: open GPU config, set Anti-aliasing → Off, Use Vulkan → Off, Anisotropic filtering → Off, and FPS target → 60. Restart RuneLite. For most players this alone fixes the crash.

🎯 When You CAN Turn Settings Up

Once your drivers are current and the plugin is stable, you can re-enable 2x-4x anisotropic filtering and 2x MSAA if you have a dedicated GPU. Only push further if a full session shows zero crashes — no setting is worth a crash mid-boss.

3. 🛡️ Update Your GPU Drivers (NVIDIA & AMD)

The single most reliable fix for a crashing GPU plugin is a clean, current driver. RuneLite talks directly to your graphics driver through OpenGL/Vulkan, so a stale or corrupted driver is the #1 reason the render context dies. Here is how to do it per manufacturer.

③ 🟢 NVIDIA (GeForce / RTX)

Step 1 — Get the latest Game Ready driver

Open the NVIDIA App (or GeForce Experience) → Drivers tab → Check for updates. Download and install the newest Game Ready Driver. Alternatively, download it manually from nvidia.com/drivers by entering your GPU model.

Step 2 — Choose "Custom" + "Clean install"

Pick Custom (Advanced) and tick Perform a clean installation to wipe old OpenGL/Vulkan profiles — a common crash source. Restart your PC afterward.

④ 🔴 AMD (Radeon / RX)

Step 1 — Update through Adrenalin

Open AMD Software: Adrenalin Edition → Settings (gear) → System → Check for updates. Install the latest driver. AMD's Vulkan driver in particular is updated frequently, so staying current matters more here than on NVIDIA.

Step 2 — Use AMD Cleanup Utility if needed

If crashes persist, run the AMD Cleanup Utility in Safe Mode to fully remove the old driver, then reinstall fresh. This fixes "worked yesterday, crashes today" cases after a half-finished update.

⑤ 🔵 Intel (Arc / Iris / UHD — common on laptops)

Intel integrated graphics rely on the Intel Arc & Iris Xe Graphics driver (or the generic Intel graphics driver for older UHD parts). Download the latest from Intel's driver site and install it. Integrated GPUs have the least memory headroom, so keep MSAA and anisotropic filtering OFF here.

⚠️ Laptop Owners: Pick the Right GPU

Many gaming laptops have two GPUs — an efficient Intel integrated chip and a discrete NVIDIA/AMD card. Windows sometimes runs RuneLite (javaw.exe) on the weak integrated GPU. Force the discrete card: Windows Settings → System → Display → Graphics, add javaw.exe (find it via Task Manager → right-click RuneLite → "Open file location"), and set it to High performance. Then restart RuneLite.

4. 💡 OpenGL vs Vulkan: Which Should You Use?

The GPU plugin can render through two backends. You switch between them with the "Use Vulkan" toggle in the GPU config. The backend you pick has the biggest impact on both FPS and stability.

FactorOpenGL (Default)Vulkan (Opt-in)
CompatibilityExcellent — works on almost everythingGood, but needs very recent drivers
StabilityVery stableCan crash, especially on AMD
PerformanceSolid, mature codebaseOften 10-30% more FPS when it works
Best forEveryone, especially laptops/integratedHigh-end desktops with stable NVIDIA drivers
Our recommendationUse this firstOnly after drivers are current and stable
💡 The Practical Rule

Leave Use Vulkan OFF unless you have a specific reason to chase extra FPS and your system is rock-solid. If you enable Vulkan and the client crashes or the plugin disables itself, switch straight back to OpenGL — the FPS gain is never worth a crash in the middle of a Vorkath kill.

5. 🐛 Known Plugin Conflicts

RuneLite plugins all share one render pipeline. When two plugins try to control it at the same time, the GPU plugin can fail to initialize or crash mid-session. These are the conflicts players report most often:

Conflict SourceEffectFix
117 HD pluginRequires the GPU plugin; disabling GPU while 117 HD is on breaks renderingKeep GPU enabled; never toggle GPU off with 117 HD active
External "RuneLite HD" / forked clientsReplaces the renderer entirely; conflicts with official GPU pluginUse only official RuneLite from runelite.net
Stretched Mode plugin + GPU stretchTwo stretch routines fight over the framebufferUse GPU plugin's built-in stretch, disable the separate plugin
Discord / Steam overlayHooks the OpenGL context and can crash itDisable in-game overlay for RuneLite
RivaTuner / MSI AfterburnerOn-screen stats inject into the contextAdd RuneLite to the exclusion list, or close the tool
OBS (certain capture modes)Game capture can destabilize the contextUse Display Capture, or run OBS as administrator

① Isolate the Culprit in Safe Mode

Hold Shift while double-clicking the RuneLite launcher (or pick "Safe Mode" if your launcher offers it). This starts RuneLite with all plugins disabled. If the GPU plugin works fine in Safe Mode, a plugin is the problem — re-enable your plugins one batch at a time until the crash returns, then remove that plugin.

🛡️ Stay on Official RuneLite

After the July 2026 bot-client shutdown, unsupported or modified clients are both risky for your account and far more likely to crash the GPU plugin. Always download RuneLite from the official site and keep your account security tight.

6. 🔄 Fallback to Software Rendering

If the GPU plugin simply will not run on your machine, you are not stuck. RuneLite automatically falls back to its built-in software renderer (your CPU draws the game). It is slower — often 15-30 FPS instead of 50+ — but it is fully playable for skilling, questing, and most content.

⑥ How to Recover After a Crash

Step 1 — Confirm the fallback is active

The game is on software rendering if it looks slightly softer and FPS dropped. This is normal and safe — keep playing or close and fix the cause.

Step 2 — Clear the corrupted shader cache

A corrupted cached shader is a known crash trigger. Close RuneLite, then delete the GPU shader cache. On Windows it lives at %USERPROFILE%\.runelite\cache\gpu — delete that gpu folder. RuneLite rebuilds it cleanly on next launch.

Step 3 — Re-enable the plugin

Update your driver (Section 3), set the safe profile (Section 2), then re-toggle the GPU plugin on in the plugin hub. If it still disables itself, leave it off and enjoy software rendering — it is a legitimate way to play.

📱 No Working GPU? Play on Mobile

If your PC's graphics are truly unusable for the session, the official OSRS Mobile app lets you continue your account from a phone or tablet with zero client setup. It is a perfect stopgap while you sort out drivers at home. (If you ever see a VPN login block, that guide walks you through clearing it.)

7. ❓ FAQ — RuneLite GPU Plugin

Why does my GPU plugin say "disabled due to an error"?

RuneLite disables the plugin automatically when the OpenGL/Vulkan render context fails — usually from an outdated driver, a too-heavy setting, or a plugin conflict. The game switches to the software renderer so you can keep playing. Update your GPU driver, set the safe profile in Section 2, and re-enable the plugin.

Is Vulkan better than OpenGL in RuneLite?

Vulkan can deliver 10-30% higher FPS on high-end desktops with current drivers, but it is less stable and crashes far more often — particularly on AMD hardware. OpenGL is the default for good reason: it works almost everywhere. Keep Vulkan off unless you specifically need the extra FPS and your system is stable.

Will using the GPU plugin get me banned?

No. The GPU plugin is an official, Jagex-approved part of the RuneLite client. It only changes how the game is drawn, not how you play. You will not be flagged for using it. What gets accounts banned is third-party "HD" clients or botting software — stick to official RuneLite.

My FPS dropped badly right after a RuneLite update. What happened?

The update likely reset or changed the GPU plugin, or a driver/Windows update broke the render context. First check whether the GPU plugin is still enabled (Configuration → GPU). If it disabled itself, follow the recovery steps in Section 6. If it is enabled but slow, your driver may need a clean reinstall.

How do I fully reset the GPU plugin to default?

In RuneLite, right-click the GPU plugin in the plugin hub and choose Reset (or toggle it off, then on). For a deeper reset, close RuneLite and delete the %USERPROFILE%\.runelite\cache\gpu folder so shaders are recompiled from scratch. Combined with a clean driver install, this resolves almost every persistent crash.
