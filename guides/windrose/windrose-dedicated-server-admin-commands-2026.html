Windrose Dedicated Server Admin Commands & How to Enable Admin

🎖️ Level: Advanced

Hosting your own Windrose dedicated server? Admin access is what turns a bare lobby into a managed world. This guide shows you exactly how to promote yourself to admin, then walks through every admin command — kick, ban, give, settime, weather, spawn and more — with syntax and copy-paste examples you can run today.

🛡️ Server Administration · 10 min read · Updated July 2026

⏱️ Quick Summary — 30-Second Read

  • 📌 Enable admin: set an AdminPassword in the server config, then run /enableadmin [password] in-game as the host or a promoted player.
  • 📌 Promote others: use /setpermission [player] admin so trusted friends can help moderate without sharing the password.
  • 📌 Moderate: /kick and /ban remove troublemakers; /unban reverses it from the ban list.
  • 📌 World control: /settime, /weather, /spawn and /give let you script events, fix stuck players, or seed resources.
  • 📌 Stay safe: keep the admin password out of public chat, log commands, and never grant admin to strangers.

1. 🔧 What You Need Before Starting

Before you can run a single admin command, your dedicated server has to be in a state where admin is actually possible. Most "commands don't work" tickets come from skipping one of these prerequisites, not from bad syntax.

① You must be running a dedicated server (not peer-to-peer)

Admin commands described here apply to a dedicated / rented Windrose server with console or config access. If you are just hosting a quick co-op session from the in-game menu, moderation is limited to the host's in-game pause menu. For true kick/ban/spawn control you need a dedicated instance — see the multiplayer guide linked below.

② You need write access to the server config

This means either SSH / file access to the server box, or the hosting panel's "Configuration" / "Startup Parameters" editor. You'll be editing a file (commonly ServerSettings.ini or the equivalent) to set the admin password.

③ You need the in-game console open

Windrose admin commands are entered through the in-game chat/console (usually by typing / then the command, or pressing ~ / F1 to open the dev console depending on your build). Keep it handy before you promote yourself.

💡 Tip

If you are migrating from a home-hosted world, back up your Save folder first. Enabling admin and running give/spawn commands modifies the live world — a backup means you can roll back if a command goes sideways.

2. ⚙️ How to Enable Admin on a Dedicated Server

Enabling admin is a two-part process: declare an admin password in the server config, then authenticate in-game. Follow these steps in order.

Step 1 — Set the Admin Password in Config

Open your server's configuration file and add (or edit) the admin password line. A typical entry looks like this:

AdminPassword=YourSecretPass123

Choose a long, unique password. Do not reuse your panel login or anything guessable. Save the file and restart the server so the new password loads.

Step 2 — Authenticate In-Game

Join the server, open the console, and run:

/enableadmin YourSecretPass123

If the password matches the config, you'll see a confirmation like "You are now an admin." From this point every admin command in section 4 will execute for you.

Step 3 — (Optional) Promote Trusted Players

Instead of handing out the password, promote specific players so they can moderate without it:

/setpermission CaptainRhea admin

They then run /enableadmin with their own session — you stay in control of the master password while they get kick/ban authority.

⚠️ Password in Chat = Compromised

Never type the raw AdminPassword into public chat. Other players can see it and lock you out of your own server. Always authenticate via the console, which is local to your client.

3. 🛡️ Admin vs. Mod Permission Tiers

Windrose uses a simple permission ladder. Knowing the tiers stops you from accidentally giving a random player full world-edit powers.

TierCan doSet with
Player (default)Play, build on claimed land, trade—
ModKick, ban, mute, view logs/setpermission [player] mod
AdminEverything a mod can, plus give, spawn, settime, weather, teleport, wipe/setpermission [player] admin
OwnerFull control incl. config-level resetsConfig AdminPassword holder

For most community servers, keep admins to 1–2 people and hand out mod to a few active, trusted regulars. That limits the blast radius if an account is shared or compromised.

Item and entity tokens vary by Windrose build. If /give Rhea oak_plank 50 fails, run /listitems (or check your server's item list) to get the exact internal name. Wrong casing is the #1 cause of "command accepted but nothing happens."

5. 🎯 Practical Admin Scenarios (Examples)

Theory is fine, but here are real situations where these commands save your server.

Scenario A — A griefer is spawn-killing newcomers

Open console, identify them, then /kick Griefer01 Stop griefing. If they return, escalate to /ban Griefer01 Repeated spawn-kill. The ban sticks across reconnects via their player ID.

Scenario B — Your community wants a night-time festival

Set the mood without waiting for the cycle: /settime 21 then /weather clear. Broadcast it: /broadcast Tonight's lantern festival begins now!

Scenario C — A new player is stuck under the deck / out of bounds

Rescue them with /tp Newbie Rhea (pull them to you) or /heal Newbie if they're taking void damage. No need to restart the session.

Scenario D — You want to seed a trading hub with starter mats

Hand a builder resources: /give BuilderBob iron_ingot 200. Pair with /save right after so the grant is persisted before anyone disconnects.

📋 Pre-event Admin Checklist

  1. Confirm you're authenticated (/enableadmin succeeded).
  2. Run /listplayers to grab exact names/IDs.
  3. Pre-stage any /give or /spawn commands in notepad.
  4. Announce with /broadcast at start and end.
  5. Finish with /save to lock in changes.

6. ⚠️ Common Mistakes & Safety Tips

MistakeWhy it bites youFix
Sharing the admin password in chatAnyone can lock you outUse /setpermission instead; keep password private
Granting admin to a strangerThey can /give themselves everything and /ban youStart with mod, promote slowly
Wrong item/entity casingCommand "succeeds" but nothing spawnsVerify tokens with /listitems
Forgetting /saveGrants lost on crashSave after every batch of edits
Over-banning for minor issuesEmpty server, bad reputation/kick + warn first, /ban as last resort
⚠️ Log Everything

Most dedicated hosts write admin actions to a log file. Review it after events — if a "mod" banned someone with no reason, you'll catch abuse early. Transparency keeps your community trusting the team.

7. ❓ FAQs

Why do my admin commands say "unknown command"?

Nine times out of ten you haven't authenticated. Run /enableadmin [password] first, and make sure the password in config matches exactly (case-sensitive). Also confirm you're on a dedicated server, not a peer-to-peer session.

Can I enable admin without restarting the server?

The password itself must be loaded at startup, so changing AdminPassword requires a restart. But once it's set, you can promote/demote players live with /setpermission without restarting.

How do I unban someone I banned by mistake?

Run /unban [player] from an admin session. If the ban was written to a ban list file, you can also remove the line there and restart. Always note the exact player ID when banning so unban is clean.

Will using admin commands flag me as a cheater?

No — admin commands are a legitimate server-management feature, not an exploit. They only affect your own dedicated world. Just don't use them on official/ranked servers you don't own.

My /give worked but the items vanished on logout — why?

You likely didn't run /save before the player disconnected or the server crashed. Get in the habit of saving after every grant. Also double-check the item token was valid; an invalid token can "succeed" with no effect.
